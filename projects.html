Battleship Unknown

This was a game that I created with my partner, Taha. We knew that we needed to make a game that was compatible on desktop and mobile. In addition, the game needed the ability to allow people to play with a friend or computer opponent. Hence why we chose to make a Battleship game with some twists. The twists were ships being able to move each round, both players don't know where their ships' locations are and each ship having their own unique ability. The game mechanic is simple which made it easier for us to make a computer opponent. After we decided what game we wanted to make, we started by creating a layout of our game in Figma. We then moved on to Lucidchart where we made a turn flowchart and a program flowchart. A flow chart allows us to visualize what happens when a condition is met. Depending on what condition is met, there will be a different output. When we finished with the flowcharts, we started programming the game logic with JavaScript. The reason we wanted to work on JavaScript first is because we wanted to make the game work before we worked on the look of the game. There was a multiple roadblock we encountered working on the game logic. One of the major road block we encounter was preventing the ship coordinates from overlapping. Our solution was to create a loop that checked for the ship's coordinates with each other for any that had the same. If it returned true, the ship would not move this round. The second to last step was to write the HTML and CSS. The final step was having people playing our game to give us feedback and find bugs. With feedbacks and bugs we made some quick fixes. Overall, this project has taught me the difficulty of time management.
